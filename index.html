node express使用的一些总结

  • node 是单线程进程，一定要做好异常处理，如果出现异常未及时处理，可能程序就会挂。当然，也可以使用node cluster启动node多进程。
  • express 的全局异常捕获，需要放在加载完routes之后，并且异步回调抛出的异常要使用next()接收,例如:next(new Error(‘xxx’))。这样异步回调抛出的异常就可以使用全局异常捕获。当然，也可以domain模块处理，不过这个模块已经被标明过时。另外，es6提出promise,es7提出async和await，来优雅处理异步回调。
  • express使用cookie-parser模块后，可以使用request.cookies获取cookie数据，获取数据格式为json格式，可以直接使用json[key]获取value。cookie-parser使用https://www.npmjs.com/package/cookie-parser
  • 如果使用node reuqest模块来发送http请求。设置http Authorization信息时,需要在options使用auth: {“bearer”:xxx}设置，直接在header设置Authorization信息的话，服务端无法获取auth token。参考https://www.npmjs.com/package/request#http-authentication
  • 使用node request模块发送http请求，需要http header注意content-type的内容设置。服务端获取application/json 与 application/x-www-form-urlencoded的方式的数据还是有很大区别的。参考https://segmentfault.com/a/1190000007252829

feign使用

1、使用场景
在微服务架构中，服务的请求者以何种方式调用远程服务是一项必须要解决的问题。在Spring Cloud(Netflix)技术栈中，每个微服务是以HTTP REST接口的形式暴露的，这样在执行远程调用时，正常情况下需要使用一个HTTP客户端，然后向服务发起HTTP请求。实际上，如果使用Netflix的Feign作为Http Client的话，我们就可以做到像Dubbo一样，服务的调用者直接调用接口方法调用远程服务，而不需要通过常规的Http Client构造请求再解析返回数据(原话出处：http://blog.csdn.net/neosmith/article/details/52180852)
使用feign还有另外一个重要原因：通过feign调用其他远程服务，只需少量的代码即可完成，并且实现了负载均衡。
2、使用
官网 http://cloud.spring.io/spring-cloud-static/Dalston.SR5/single/spring-cloud.html#spring-cloud-feign
别人整理 http://blog.csdn.net/Hzt_fighting_up/article/details/78248720
3、源码解析，参考 http://blog.csdn.net/forezp/article/details/73480304
4、个人实现的简单学习demo https://github.com/wlfewn/study-demo/tree/master/cloud-session-feign-demo
5、问题总结
1.如果http请求的header信息无法传递，参考 http://blog.csdn.net/u014519194/article/details/77160958
2.对于需要使用@FeignClient的项目，一定要使用@EnableFeignClients显式声明,通过源码可知
3.同时使用spring session与feign，使用feign的项目同样需要引入spring session，否则会出现一种每次session id不一致的情况

WebSocket学习整理

基础概念
简单的小例子,使用spring boot搭建的后台
WebSocket下客户端身份确认,实现思路
1、客户端通过用户名密码调用服务端的接口来获取角色Token
2、链接websoket时带上此Token（ws://localhost:8080?token=Token）
3、websoket服务端建立连接时验证此Token是否有效，无效则连接失败
4、有效，通过Token中的数据来设置此连接的一些信息（比如：用户ID类似的）
5、以后客户端发送请求使用此连接
