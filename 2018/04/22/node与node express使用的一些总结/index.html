node express使用的一些总结

  • node 是单线程进程，一定要做好异常处理，如果出现异常未及时处理，可能程序就会挂。当然，也可以使用node cluster启动node多进程。
  • express 的全局异常捕获，需要放在加载完routes之后，并且异步回调抛出的异常要使用next()接收,例如:next(new Error(‘xxx’))。这样异步回调抛出的异常就可以使用全局异常捕获。当然，也可以domain模块处理，不过这个模块已经被标明过时。另外，es6提出promise,es7提出async和await，来优雅处理异步回调。
  • express使用cookie-parser模块后，可以使用request.cookies获取cookie数据，获取数据格式为json格式，可以直接使用json[key]获取value。cookie-parser使用https://www.npmjs.com/package/cookie-parser
  • 如果使用node reuqest模块来发送http请求。设置http Authorization信息时,需要在options使用auth: {“bearer”:xxx}设置，直接在header设置Authorization信息的话，服务端无法获取auth token。参考https://www.npmjs.com/package/request#http-authentication
  • 使用node request模块发送http请求，需要http header注意content-type的内容设置。服务端获取application/json 与 application/x-www-form-urlencoded的方式的数据还是有很大区别的。参考https://segmentfault.com/a/1190000007252829
